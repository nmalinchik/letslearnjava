1.2 Исполняемые Java-приложения

Для успешного получения сертификата OCA Java SE 8 Programmer I необходимо понимать структуру и назначение java-приложений, а также понимать, как именно обычный java-класс превращается в исполняемый.

Также вы должны знать, как запустить Java-приложение из командной строки.

1.2.1 Исполняемые Java-классы против неисполняемых

Вы спросите:"А разве не виртуальная машина Java (JVM) выполняет все java-классы, когда они используются? Если да - то что тогда такое неисполняемые классы?"

Исполняемые Java классы, попадая в руки к JVM, начинают свое выполнение с определенного места в классе, а именно метода main. JVM начинает выполнять код, который находиться в main-методе. К сожалению, заставить бездушную JVM исполнить неисполняемый класс без main-метода не получиться, потому как JVM не будет знать с какого метода ей начинать выполнение.

В стандартном Java-приложении содержится n-нное количество классов и интерфейсов, раскиданные по java-файлам. И среди них, как правило, есть всего один-единственный исполняемый класс. Именно в этом классе разработчик размещает логику, по которой должны вызываться методы и классы, которые в последствии выполняет JVM, чтобы приложении корректно работало.

1.2.2 Main-метод

Главным требованием при создании исполняемого Java-приложения является создание класса с методом main, сигнатура которого (название и аргументы) совпадают с примером ниже:

public class Example{

public static void main(String args[]){

System.out.println("Hello!");

}

}

main-метод должен определяться по следующим принципам:

  • Модификатор доступа public
  • Метод должен быть статическим (static)
  • Имя метода main (с маленькой буквы)
  • Возвращаемый тип void (то есть метод ничего не возвращает)
  • Аргументом метода может быть либо массив строк String array (String args[]) либо список строк переменной длины varargs (String... args).

То есть код ниже тоже очень даже валидный:

public static void main(String… args){}

При этом допускается использовать любое имя переменной для массива (String cats[] к примеру вполне сойдет).

Скобки массива тоже можно ставить где пожелаете: и после типа и после названия - оба варианта допустимы (String cats[], String[] cats).

Очередность "public static" также не имеет значение, можете конечно писать "static public", но, боюсь, что на вас могут косо смотреть ваши коллеги).

ДОБАВИТЬ ССЫЛКУ!!!!!!!!!!!!!!

В случае, если сигнатура метода будет отличаться от требований, указанных выше, данный метод станет перегруженным (overloaded). Перегруженным методом называются методы с одинаковыми названиями, но разной сигнатурой (более подробно, мы рассмотрим данный вопрос в уроке 3).

Для понимания о чем идет речь, рассмотрим пример:

public class Example {

public static void main(String args) {

System.out.println("В параметре одна строка");

}

public static void main(String args[]) {

System.out.println("Этот метод и выполнит JVM");

}

public static void main(int number) {

System.out.println("Параметр в виде числа");

}

}

1.2.3 Запуск Java приложения из командной строки

Сейчас очень сложно найти староверов, которые не пользуются прелестями IDE (Integrated Development Environment интеграционными системами разработки).

Однако, при сдаче экзамена необходимо знать и понимать как запустить java-приложение из командной строки. Если вы не помните или не знаете как это делать, дабы освежить память Oracle предлагает пройти небольшой курс.

Давайте рассмотрим пример на практике: у нас есть простой файл Example.java с классом:

public class Example{

public static void main(String args[]){

System.out.println("Hello!");

}

}

Чтобы выполнить код, написанный в классе выше, в командной строке необходимо перейти в папку с этим файлом и выполнить команду java Example, после выполнения кода, в консоль выведется "Hello!".

Для того, чтобы передать параметры в main-метод в пример ниже:

public class Example{

public static void main(String args[]){

System.out.println args[0]);

System.out.println args[1]);

}

}

В консоли необходимо ввести следующую команду java Example 1 2, где 1 это первый элемент в массиве args, а 2 второй (но так как в java-массивах нумерация начинается с 0, соответственно 1 - будет находиться в массиве в ячейке № 0, 2 - в ячейке № 1).

В консоль выведется 2 строки "1" в первой "2" во второй.

Именно потому, что параметры в массив args попадают из командной строки, они и называются параметрами командной строки.
