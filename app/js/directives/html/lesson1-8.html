1.8 Краткая шпаргалка по теме

Структура и компоненты Java класса:

  • Экзамен OCA Java SE 8 Programmer I exam включает вопросы по структуре и компонентам java-классов и java-файлов и не включает вопросы по файлам с байткодом (.class файлы).
  • Класс может включать много компонентов: операторы import и package, переменные, конструкторы, методы, комментарии, вложенные классы и интерфейсы, аннотации и перечисления (enum).
  • Экзамен не затрагивает внутренние классы и интерфейсы, аннотации и перечисления.
  • Если в классе есть оператор package, то он должен быть первым оператором при определении класса.
  • Оператор package не может использоваться без определения класса, а также после определения класса.
  • Оператор package может быть использован максимум один раз в классе (может и не использоваться).
  • Оператор import позволяет использовать короткие имена классов и интерфейсов.
  • Нельзя импортировать несколько классов или интерфейсов с одинаковыми именами.
  • В классе может быть несколько импортов.
  • Если в классе есть оператор package, то оператор import должен идти сразу после него.
  • Если оператор import используется, то он должен идти перед определением класса или интерфейса.
  • Комментарии используются для комментирования кода и могут быть использованы в нескольких местах в классе.
  • Комментарии могут быть размещены до и после оператора package (единственное исключение), до и после определения класса, до, в и после методов.
  • Есть два типа комментариев: многострочные и комментарий до конца строки.
  • В комментариях разрешается использовать любые спецсимволы.
  • Многострочные комментарии могут включать от 1 и более строк. Они начинаются с /* и заканчиваются ш.
  • Комментарии до конца строки начинаются с // и распространяются до конца строки. Текст между // и концом строки является комментарием.
  • В классе может как не содержаться вовсе, так и содержаться сколько угодно переменных, методов и конструкторов.
  • Порядок определения переменных, конструкторов и методов в классах не имеет значения.
  • Переменная, определенная до или после метода в классе, может быть использована в этом методе без возникновения ошибок.
  • В java-файлах может быть определено от одного и более классов и интерфейсов.
  • Публичный класс может быть определен только в java-файле с таким же именем.
  • Операторы package и import применяются ко всем классам и интерфейсам, определенным в одном и том же java-файле.

Исполняемые Java приложения:

  • Исполняемые классы — это классы, которые имеют точку входа для Java Virtual Machine (JVM). Этой точкой является метод main.
  • Для того, чтобы класс стал исполняемым, в нем должен быть метод main со следующими сигнатурами: public static void main(String args[]) или public static void main(String... args). Очередность операторов public и static может взаимозаменяемым, а имя массива может быть любым (args можно заменить на любое валидное имя).
  • В классе может быть несколько методов main с тем условием, что их сигнатуры отличаются от тех, которые были указаны в предыдущем пункте. Они будут называться перегруженными и не будут считаться методом main.
  • Метод main принимает массив строк в виде параметров, передаваемые JVM.
  • При запуске класса из командной строки, слова java и имя класса не являются параметрами, передаваемыми в метод main.

Оператор package:

  • Вы можете использовать оператор package для объединения в группы связанные классы и интерфейсы.
  • По умолчанию все классы и интерфейсы, которые находятся в разных пакетах, невидимы друг для друга.
  • Имя пакетов и подпакетов разделяются точкой.
  • Все пакеты и интерфейсы, содержащиеся в одном пакет, видны друг другу.
  • Оператор import позволяет использовать короткие имена классов и интерфейсов, находящихся в других пакетах.
  • В одном класса нельзя импортировать несколько классов и интерфейсов с одинаковыми именами, находящихся в разных пакетах.
  • Вы можете импортировать один член пакета или все элементы из пакета, используя знак * после указания подпакета (import edu.mycompany.test.*;)
  • Классы и интерфейсы, которые находятся в пакете по умолчанию, нельзя использовать в классах, находящихся в именованных пакетах, даже если классы находятся в одной и той же директории.
  • Вы можете импортировать отдельны статический элемент класса или сразу все статические элементы используя оператор static import.
  • Оператор import не может быть размещен перед оператором package в классе. Любая попытка карается ошибкой компиляции.
  • Члены пакета по умолчанию доступны только для классов и интерфейсов, которые находятся в той же директории (и не входят в именованный пакет).

Модификаторы доступа:

  • Модификаторы доступа позволяют ограничивать видимость ваших классов и их составляющих за пределом класса или пакета.
  • В Java есть 4 уровня доступа: public, protected, default и private.
  • В Java есть 3 модификатора доступа: public, protected и private.
  • Самым открытым является модификатор public.
  • Классы и интерфейсы с модификатором public для несвязанных и наследующихся классов как в рамках пакета, так и за его пределами.
  • Элементы класса с модификатором protected доступны для всех классов и интерфейсов в рамках одного пакета и в производных классах за пределами пакета.
  • Элементам класса без явно указанного модификатора доступа присваивается модификатор по умолчанию (еще его называют package-private).
  • Элементы класса с модификатором доступа по умолчанию доступны только для классов и интерфейсов в рамках одного пакета.
  • Классы и интерфейсы с модификатором доступа по умолчанию также недоступны за пределами пакета.
  • Элементы класса с модификатором private доступны только в рамках класса, в которых они определены. В независимости от того связаны ли классы и находятся в одном или разных пакетах, приватные поля недоступны вне класса.
  • Модификатор private – самый закрытый.

Другие модификаторы:

  • Данные модификаторы изменяют у классов и их членов свойства по умолчанию.
  • В экзамене будут вопросы только по трем из них: abstract, final и static.
  • Слово abstract, при добавлении к определению класса, делает этот класс абстрактным, даже, если в классе нет не одного абстрактного метода.
  • Объект абстрактного класса создать невозможно.
  • Интерфейсы являются абстрактными по умолчанию. Java-компилятор автоматически добавляет слово abstract к определению интерфейса (то есть явное написание слова abstract при определении интерфейсов является излишним.
  • Абстрактные методы не имеют тела. Все неабстрактные производные от абстрактного классы должны реализовывать абстрактные методы.
  • Слово abstract не используется с переменными.
  • Модификатор static может быть использован с внутренними классами и интерфейсами, переменными и методами. Внутренние классы и интерфейсы в экзамене не затрагиваются.
  • Метод не может быть одновременно статическим и абстрактным.
  • Статические методы и переменные являются общими для всех объектов этого класса.
  • Статические переменные существуют независимо от количество существующих объектов данного класса (даже если их нет вовсе).
  • Статические переменные и методы принадлежат классу, а не объекту класса – это нужно запомнить.
  • Статические переменные или методы могут быть вызваны с использованием имени класса или переменной на класс.
  • Статические методы и переменные не могут использовать не-статические методы или переменные класса. Последние, в свою очередь, могут использовать статические элементы без проблем.
  • Статические классы и интерфейсы относятся к внутренним классам. Но они не затрагиваются в экзамене, поэтому нами не рассматривались.
  • Модификатор static нельзя использовать с классами и интерфейсами, которые не являются внутренними (не находятся внутри других классов или интерфейсов).

Функции и компоненты языка Java:

  • Объектно-ориентированность – Java эмулирует характеристики и поведение реальных объектов через классы, интерфейсы, перечисления и т.д.
  • Абстракция – Java позволяет вам использовать только те свойства и поведение объектов в коде, которое вам необходимо для решения задачи.
  • Инкапсуляция – вы можете ограничить доступ к объектам класса от нежелательных изменений.
  • Наследование – Java позволяет своим классам наследоваться от других классов, а также реализовывать интерфейсы. Интерфейсы могут наследоваться от других интерфейсов.
  • Полиморфизм - это возможность применения одноименных методов с одинаковыми или различными наборами параметров в одном классе или в группе классов, связанных отношением наследования.
  • Типобезопасность – прежде, чем использовать переменную в Java, вы обязаны указать ее тип.
  • Автоматическое управление памятью – Java использует сборщик мусора для автоматической очистки памяти. Он очищает память от объектов, которые больше не используются.
  • Многопоточность – в Java есть ряд функций для разработки многопоточных приложений.
  • Java не является однопоточным языком программирования.
