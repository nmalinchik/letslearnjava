1.1.1 Структура Java класса

Экзамен OCA Java SE 8 Programmer включает вопросы по структуре и компонентам .java файлов и классам и интерфейсам, которые они содержат.

Java класс может содержать следующие компоненты:

  • Package (пакеты)
  • Import (импорты)
  • Comments (комментарии)
  • Class declaration (определение класса) {
  • Variables (переменные)
  • Constructors (конструкторы)
  • Methods (методы)
  • Nested classes (вложенные классы)
  • Nested interfaces (вложенные интерфейсы)
  • Enum (перечисления)

Последние 3 компонента не включены в программу экзамена, поэтому данные компоненты рассматриваться не будут.

Package

Все Java классы принадлежат к тому или иному пакету. Классы могут принадлежать как именованному пакету, если это явно указано, так и к пакету по умолчанию (default package), если пакет не указан.

Предназначение пакетов в том, чтобы четко разграничить структуру проекта. Определение пакета (если оно присутствует в классе), должно идти первым элементом в классе (единственное исключение, это комментарии):

package exam; //оператор package должен быть первым оператором в классе

class Test{

}

Также нужно помнить, что в классе может присутствовать только одно определение пакета.

Import

Классы и интерфейсы в одном пакете могут использовать друг друга по именам без указания имени пакета. Но для использования класса или интерфейса из другого пакета вы должны использовать их полное имя, а именно «имяПакета.имяПодпакета.имяКласса». К примеру для класса String полное имя будет java.lang.String.

По причине того, что код с полными именами будет громоздким и трудночитаемым вы можете использовать импорт, чтобы пользоваться простыми именами классов и интерфейсов.

К примеру, если взять класс String и представить, что пакет java.lang не импортируется по умолчанию (а он импортируется), для того, чтобы пользоваться  классом String по простому имени, код бы выглядел следующим образом:

package example;

import java.lang.String;

class SomeClass{

                String s = “Hello”;

}

Блок импортов следует вторым после объявления имени пакета, или же первым, если пакет не указывается (default).

В классе может быть сколько угодно импортов.

Comments

Вы можете комментировать свой код. Причем комментарии можно добавить и перед оператором package (комментарии - это единственное исключение из правил), и в любом другом месте класса.

Комментарии могут быть 2х типов – многострочные и однострочные (End-of-line).

Многострочные комментарии начинаются с /* и заканчиваются */. Пример:

Class Example{

/*

some

Comment

*/

}

В коде часто можно встретить, как в многострочных комментариях каждая строчка начинается с «*», но это необязательное требование, так делают только для красоты.. Пример:

/*

* some

* Comment

*/

Однострочные (End-of-line) комментарии начинаются с «//» и заканчиваются концом строки. Пример:

Class Example{

                String name; //name of person

}

В комментариях можно использовать любые спецсимволы (&%^*{}|\|:;"?/>.<,!@#$%^&*()).

Отдельно стоит упомянуть  Javadoc комментарии. Это специальные комментарии, которые начинаются с «/**» и заканчиваются «*/». Данные комментарии используются JDK утилитой Javadoc для генерации API документации.

Class declaration

Оператор class показывает где начинается класс. Три минимальные составляющие при определении класса – это слово class (!со строчной буквы), имя класса и фигурные скобки:

class Person {} //минимум при создании класса

Определение класса дополнительно может включать следующие элементы:

  • Модификаторы доступа
  • Другие модификаторы
  • Имя класса
  • Имя родительского класса, от которого класс наследуется
  • Имя имплементируемого/ых интерфейса/ов
  • Тело класса (поля, методы, конструкторы), размещенные между двумя фигурными скобками {}.

Пример объявления класса со всеми указанными элементами:

public final class Runner extends Person implements Athlete {}

Class definition

Класс — это шаблон, по которому создается объект. Его свойства определяются переменными, а поведение – методами. По одному шаблону можно создавать любое количество объектов.

Пример простого класса:

class Car{

String model;

int speed;

String color;

Car(String model, String color){

this.model = model;

this.colot = color;

}

void drive(){

//some code

}

void setSpeed(int speed){

//some code

}

}

Variables

Рассмотрим переменные из класса Car выше. Свойства model, speed, color являются переменными экземпляра, т.к. принадлежат конкретному объекту. У каждого объекта есть свои собственные переменные, принадлежащие только ему. Если вы поменяете значения в одной переменной, принадлежащей какому-либо из объектов этого класса, значения этой переменной в других объектах не поменяется.

Также существуют переменные, принадлежащие классу. Это статические (static) переменные. Но о них мы поговорим позже.

Methods

По тому же принципу делятся и методы. Могут быть методы конкретного экземпляра, а могут быть статические методы.

Constructors

Конструктор используется для создания объекта данного класса. В одном классе может быть несколько конструкторов с разными параметрами.

 

1.1.2 Структура и компоненты .java файла

 

.java-файлы необходимы для хранения классов, интерфейсов, enum’ов и аннотаций(аннотации не входят в программу экзамена, поэтому рассматриваться не будут).

Весь ваш Java код должен находиться в файлах с расширением .java. В экзамене затрагиваются следующие особенности структуры java-файлов:

  • Определение класса и интерфейсов
  • Определение одного или нескольких классов или интерфейсов в одном файле
  • Применение операторов import и package ко всем классам в одном файле.

Со структурой класса мы разобрались ранее, поэтому рассмотрим структуру интерфейсов.

Интерфейсы определяют ряд методов и констант, которые класс должен реализовать. До Java 8 все методы в интерфейсах были абстрактными. Однако, начиная с Java 8 методы в интерфейсах могут иметь реализацию по умолчанию (default). Также появилась возможность использовать статические методы.

Чтобы понимать назначение интерфейсов давайте рассмотрим пример: у каждого дома есть телевизор. Так вот, у всех телевизоров есть ряд общих функций: переключение каналов, включение, выключение, и т.д. Данные общие опции можно вынести в отдельный интерфейс и в последствии при создании любого класса телевизора можно будет реализовать в своем классе данные общие методы (определяющие основные функции), реализуя данный интерфейс:

interface Controls {

void changeChannel(int number);

void powerOn();

void powerOff();

}

Определение интерфейса начинается со слова interface (!с маленькой буквы)

Детально интерфейсы мы рассмотрим в уроке №6.

Один .java-файл может содержать один или несколько классов, интерфейсов, enumов или аннотаций. Ниже представлен пример с объявлением 2х интерфейсов и 2х классов в одном файле MyClass.java:

interface Printable {

//.. we are not detailing this part

}

public class MyClass {

//.. we are not detailing this part

}

interface Movable {

//.. we are not detailing this part

}

class Car {

//.. we are not detailing this part

}

В одном Java файле может быть только один публичный класс или интерфейс, причем название данного класса/интерфейса должно совпадать с именем файла (MyClass в примере выше), иначе ваш код не скомпилируется.

Что касается package и import – и то и другое указывается в начале файла и применяется ко всем классам/интерфейсам/enum/аннотациям в файле и используется по принципам, рассмотренным ранее:

// contents of Car.java

package com.manning.code;

import com.manning.*;

interface Printable {}

interface Movable {}

public class Car {}
