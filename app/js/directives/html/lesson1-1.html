1.1.1 Структура Java класса

Экзамен OCA Java SE 8 Programmer включает вопросы по структуре и компонентам .java файлов и классам и интерфейсам, которые они содержат.

Java класс может содержать следующие компоненты:

  • Package
  • Import
  • Comments
  • Class declaration
  • Variables
  • Constructors
  • Methods
  • Nested classes
  • Nested interfaces
  • Enum

Последние 3 компонента не включены в программу экзамена, поэтому данные компоненты рассматриваться не будут.

Package

Все Java классы принадлежат к тому или иному пакету. Классы могут принадлежать как именованному пакету, если это явно указано, так и к пакету по умолчанию (default package), если пакет не указан.

Предназначение пакетов в том, чтобы четко разграничить структуру проекта. Оператор package (если оно присутствует в классе) должен идти первым элементом в классе (единственное исключение, это комментарии):

package exam; //оператор package должен быть первым оператором в классе

class Test{

}

Также нужно помнить, что в классе может присутствовать только один оператор package.

Import

Классы и интерфейсы в одном пакете могут использовать друг друга по короткому имени без указания пакета. Но для использования класса или интерфейса из другого пакета нужно использовать его полное имя («имя_пакета.имя_подпакета.имя_класса»). К примеру для класса String полное имя будет java.lang.String.

По причине того, что код, где все классы и интерфейсы будут вызыватся по полным именам, станет громоздким и почти нечитаемым, был придуман оператор import, который позволяет пользоваться короткими именами классов и интерфейсов.

К примеру, возьмем класс String и представим, что пакет java.lang не импортируется по умолчанию (а он импортируется, если вы не знали). Для того, чтобы воспользоваться классом String по короткому имени, нужно было бы написать следующий код:

package example;

import java.lang.String;

class SomeClass{

                String s = “Hello”;

}

Блок импортов следует вторым после объявления имени пакета, или же первым, если пакет не указывается (default).

В классе может быть сколько угодно импортов.

Comments

Вы можете комментировать свой код. Даже не так, я вам очень рекомендую комментировать свой код (поможете и себе и коллегам).

Комментарии можно добавлять в любом месте в коде. Да-да, в любом, и перед оператором package в том числе. Касательно оператора package, комментарии - это единственное исключение из правил.

Комментарии могут быть 2х типов: многострочные и однострочные (End-of-line).

Многострочные комментарии начинаются с /* и заканчиваются */. Пример:

Class Example{

/*

some

Comment

*/

}

Часто в импортируемых классах можно встретить, что в многострочных комментариях каждая строчка начинается со значка звездочки *, но это необязательное требование, это делают сугубо для красоты и эстетичности (ну чтобы все по фен-шую было). Пример:

/*

* some

* Comment

*/

Однострочные (End-of-line) комментарии начинаются с «//» и заканчиваются концом строки. Пример:

Class Example{

                String name; //name of person

}

В комментариях можно использовать любые спецсимволы (&%^*{}|\|:;"?/>.<,!@#$%^&*()).

Отдельно стоит упомянуть  Javadoc комментарии. Это специальные комментарии, которые начинаются с: /**» и заканчиваются:*/. Данные комментарии используются JDK утилитой Javadoc для генерации API документации.

Class declaration

Оператор class показывает начало класса. Тремя минимальными элементами при определении класса являются: слово class(!со строчной буквы), имя класса и фигурные скобки {}:

class Person {} //этого достаточно для создания класса

Определение класса также может включать следующие необязательные элементы:

  • Модификаторы доступа
  • Другие модификаторы
  • Имя родительского класса, от которого класс наследуется
  • Имя имплементируемого/ых интерфейса/ов
  • Тело класса (поля, методы, конструкторы), размещенные между двумя фигурными скобками {}.

Пример объявления класса со всеми указанными элементами:

public final class Runner extends Person implements Athlete {}

Class definition

Класс - это шаблон, по которому создается объект. Его свойства определяются переменными, а поведение - методами. По одному шаблону можно создавать любое количество объектов.

Пример простого класса:

class Car{

String model;

int speed;

String color;

Car(String model, String color){

this.model = model;

this.colot = color;

}

void drive(){

//some code

}

void setSpeed(int speed){

//some code

}

}

Variables

Рассмотрим переменные из класса Car выше. Свойства model, speed, color являются переменными экземпляра, т.к. принадлежат конкретному объекту. У каждого объекта есть свои собственные переменные, принадлежащие только ему. Если вы поменяете значения в одной переменной, принадлежащей какому-либо из объектов этого класса, значения этой переменной в других объектах не поменяется.

Также существуют переменные, принадлежащие классу. Это статические (static) переменные. Но о них мы поговорим позже.

Methods

По тому же принципу делятся и методы. Могут быть методы конкретного экземпляра, а могут быть статические методы.

Constructors

Конструктор используется для создания объекта данного класса. В одном классе может быть несколько конструкторов с разными параметрами.

1.1.2 Структура и компоненты .java файлов

.java-файлы необходимы для хранения классов, интерфейсов, enum'ов и аннотаций (аннотации не входят в программу экзамена, поэтому рассматриваться не будут). Весь ваш Java-код должен находиться в файлах с расширением .java.

В экзамене будт затронуты следующие особенности структуры java-файлов:

  • Определение класса и интерфейсов
  • Определение одного или нескольких классов или интерфейсов в одном файле
  • Применение операторов import и package ко всем классам в одном файле.

С общей структурой класса мы разобрались ранее, поэтому сейчас рассмотрим структуру интерфейса.

Интерфейсы определяют ряд методов и констант, которые класс должен реализовать. До Java 8 все методы в интерфейсах были абстрактными. Однако, начиная с Java 8, методы в интерфейсах могут иметь реализацию по умолчанию (default). Также появилась возможность использовать статические методы.

Чтобы понять предназначение интерфейсов давайте рассмотрим такой пример: у каждого из нас дома есть телевизор (ну или монитор, планшет, без разницы). Так вот, у всех телевизоров можно выделить ряд общих функций: переключение каналов, включение, выключение, и т.д. Данные функции можно вынести в отдельный интерфейс для того, чтобы в последствии при создании конкретной модели телевизора можно было имплементировать этот интерфейс и реализовать данные общие функции в своем классе (а то что это за телек, которой включить то нельзя?).

Данный интерфейс можно представить следующим образом:

interface Controls {

void changeChannel(int number);

void powerOn();

void powerOff();

}

Чтобы создать интерфейс, заменяем слово class на interface (!с маленькой буквы)

Детально интерфейсы мы рассмотрим в уроке №6.

Один .java-файл может содержать один или несколько классов, интерфейсов, enum'ов или аннотаций. Ниже представлен пример .java-файла "MyClass.java" с объявлением в нем 2х интерфейсов и 2х классов:

interface Printable {

//.. какой-то код

}

public class MyClass {

//.. какой-то код

}

interface Movable {

//.. какой-то код

}

class Car {

//.. какой-то код

}

В одном .java-файле может быть только один публичный класс или интерфейс, причем название данного класса/интерфейса должно совпадать с именем файла (MyClass в примере выше), иначе ваш код не скомпилируется.

Что касается package и import - и то и другое указывается в начале файла и применяется ко всем классам/интерфейсам/enum/аннотациям в файле (к ним применяются общие правила, рассмотренным ранее):

// contents of Car.java

package com.manning.code;

import com.manning.*;

interface Printable {}

interface Movable {}

public class Car {}
